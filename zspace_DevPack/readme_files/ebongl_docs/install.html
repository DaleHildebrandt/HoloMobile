Installation



Installation guides are provided for the following platforms:

  • Linux Instructions
  • Visual Studio Instructions
  • Using EbonGL

Linux Instructions

These instructions should work on most Linux distributions.


0. Requirements

  • g++ compiler version 4 or later
  • OpenGL development files, version 1.2 or later
  • GLUT development files
  • MathHelper library
  • Open Asset Import Library (Assimp)
  • Developer's Image Library (DevIL)

Under Ubuntu, you can get g++ by installing the build-essential package. OpenGL and GLUT may be already installed along with your video card drivers, otherwise you can install them using the open-source packages mesa-common-dev and libglu1-mesa-dev.


1. Compiling

Compiling is as simple as:

  • cd ebongl_directory
  • ./configure
  • make
  • make install

If you would like to install somewhere other than the default location, you can pass --prefix=/my_location to the configure script. The last step, make install, should be performed as root.


2. Linking

Link your program to EbonGL, MathHelper and the math library:

g++ MyProgram.cpp -lebongl -lmathhelp -lm

See the Assimp documentation for linking to Assimp and DevIL.


Visual Studio Instructions

These instructions were tested on Windows 7.


0. Requirements

  • Visual Studio 2010 or later
  • OpenGL development files, version 1.2 or later
  • GLUT development files
  • MathHelper library
  • Open Asset Import Library (Assimp)
  • Developer's Image Library (DevIL)

By default, EbonGL expects to find the MathHelper, DevIL and Assimp libraries in C:\tools.

  • MathHelper C:\tools\hcilibs_64bit_vs2010
  • Assimp C:\tools\assimp_64bit_vs2010
  • DevIL C:\tools\devil_64bit_vs2010

Each folder should contain the following subdirectories:

  • A \lib subdirectory for the library's .lib files
  • An \include directory for the library's .hpp files


1. Building

A Visual Studio solution is provided for building EbonGL. Two files should be created:

  • Release Mode: libebongl.lib
  • Debug Mode: libebongl_d.lib

The .lib files should be copied to your library folder. EbonGL's libebongl subdirectory should be copied to your /include folder, as well as EbonGL.hpp.


Using EbonGL

Include

To use EbonGL, include EbonGL.hpp:

#include "EbonGL.hpp"


Namespace

All classes, enumerations and constants belong to the EbonGL namespace. They can be accessed either by adding the namespace:

using namespace EbonGL;

or by specifying the scope:

EbonGL::EG_Color myColor;
