Class EG_ConfigWriter



A simple configuration file manager, intended to make it easy to hand edit the file, or to have the program update configuration values.

File Format:

  • Each entry in the file has 0 or more comments, an identifier, and a value
  • Comments start with a #, IDs start with a %, and the value starts with a >
  • There should be no spaces between lines in a given entry

Example: 

  #This is my descriptive comment
  #Comments work great
  %My Favorite Color
  >Blue

By using the key "My Favorite Color", we can load the value Blue from the file, or change its value, with one line of code.

A convenience function that converts number to an EG_String.

static double loadDouble(const EG_String &fileName, const EG_String &configID)

static int loadInt(const EG_String &fileName, const EG_String &configID)

static EG_String loadString(const EG_String &fileName, const EG_String &configID)

These functions load a value from file. fileName is the name of the file to read from. configID is a unique identifier for the value we want to read. For example, "My Favorite Color". configID is case-sensitive. If the file couldn't be read, or if configID wasn't found, these functions will throw an exception.


static void setPath(const EG_String &newPath)

Set the path to the directory containing the config file(s). By default, the path is the current directory. newPath should end with a /.


static bool strToInt(const EG_String &text, int &theInt)

Utility function to convert a string into an integer. Returns false if text did not contain a valid integer.

static void writeInt(const EG_String &fileName, const EG_String &configID, int newInt)

static void writeString(const EG_String &fileName, const EG_String &configID, const EG_String &newString)

These functions write a value to a configuration file. fileName is the file to write to, and configID is a unique identifier for this value. For example, "My Favorite Color". configID is case-sensitive. If configID wasn't found, a new entry will be created for it.
